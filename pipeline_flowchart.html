GeoPipeline v8 当前流程图

对应当前仓库代码：geo_pipeline/pipeline.py、evaluate.py、modules/sl.py、modules/dst.py、modules/pomdp.py 和 wait_and_run.sh。

一句话版：每张图按 country → city → street 粗到细推理；每层先让 MLLM 生成候选，再用 POMDP 选检查任务、Verify 取证、SL 给证据打分、Bayes 更新 posterior，最后由评估脚本 geocode 成坐标并算距离阈值准确率。

Country posterior 是主控 City/Street 受 top candidates 约束 v8 修复 child-country 冲突 分析重点是 continent false positives

当前实验背景

v6b 的主要问题是北美假阳性：很多欧洲/亚洲图被判成美国/加拿大。v7 把北美假阳性从 825 降到 120，但引入了 51.84% 的 country-child conflict。v8 的目标是保留反北美偏置，同时阻止 city/street 随便推翻 country。

1. 全局主流程

YFCC4K Batch

evaluate.py 从 YFCC4KDataset 读取图片和 GT 经纬度，按 batch_size 送入 pipeline。

Country Level

生成国家候选，形成 country_posterior。v8 要至少做一次验证，不能 step=0 因初始概率略高就下钻。

City Level

上下文带 top-3 国家候选，要求城市在这些候选国家内细化，除非图中直接出现别国线索。

Street Level

上下文带 top 国家和 top 城市，输出街道、区、地标。v8 不鼓励附加候选外国家。

Geocode

优先 street/city 带 country 查询；如果 child 自带国家和 parent 冲突，跳过该 child，回退到更稳的层级。

Metrics

用 Haversine 距离算 1/25/200/750/2500 km 准确率，并保存 records 供诊断。

2. 单个层级内部：Hypothesize → Verify → Update

Hypothesize

MLLM 看图生成 JSON：hypotheses 和 verification_plan。

位置：_hypothesize_prompt

Parse & Normalize

解析普通 JSON、Markdown fenced JSON、以及 {general, cue_responses} wrapper。

位置：_parse_hypothesis_payload

Prior

国家层做 country canonicalization，再用 temperature softmax 得到初始 prior。

位置：_softmax_prior

POMDP Loop

只要没满足停止条件，就选任务、取证、打分、更新 posterior。

位置：_run_level

MAP Output

每层取 posterior 最大的 location，写入 country/city/street。

MAP = maximum a posteriori
现在代码不是论文里的 WebSearch/ImageSearch 版本。所有证据都来自同一个 MLLM 对图片的再次观察和解释，所以多循环可能增强正确线索，也可能重复放大偏见。后续 v9 更适合加 margin stop、replace/backtrack，而不是盲目增加循环次数。

3. POMDP + SL + Bayes 更新细节

这一节对应 GeoPipeline._run_level() 和 GeoPipeline.predict_batch() 里的核心循环。每个层级 country/city/street 都走同一套逻辑：当前 posterior 决定下一步检查什么，检查结果再反过来更新 posterior。

POMDP Select

给当前 belief 和 pending tasks，让 MLLM 选最能减少不确定性的检查任务。

代码：POMDPModule.select_action / batch 中调用 _make_policy_prompt

Verify

MLLM 按任务看图，输出一段 observation，例如文字、建筑、道路、植被如何支持/反驳候选。

代码：_verify_prompt + mllm.batch_generate

SL Score

对每个 hypothesis 采样 SL_N_SAMPLES=5 次，解析 Support Rating 和 Confidence。

代码：SLModule.score_many

Bayes / DST

默认用 GeoBayes Eq.7 乘法更新；证据高度冲突时用 Yager DST fallback。

代码：DSTModule.fuse

单图版代码骨架

posterior = dict(initial_posterior)
pending = list(initial_plan)
evidence_scores_all = []

while True:
    exhausted = len(pending) == 0
    if self.pomdp.should_stop(posterior, step, level, exhausted):
        break

    task_idx = self.pomdp.select_action(posterior, pending, level, step)
    task = pending.pop(task_idx)

    evidence_desc = self.mllm.generate(_verify_prompt(...))
    w_scores = self.sl.score(evidence_desc, hyps, level)
    evidence_scores_all.append(w_scores)
    posterior = self.dst.fuse(initial_posterior, evidence_scores_all)

Batch 版代码骨架

active = [i for i in range(n)
          if not should_stop(posteriors[i], steps[i], ...)]

policy_resps = self.mllm.batch_generate(policy_msgs)
verify_resps = self.mllm.batch_generate(verify_msgs)

sl_items = [(evidence_descs[i], list(posteriors[i].keys()))
            for i in active]
sl_results = self.sl.score_many(sl_items, level)

for k, i in enumerate(active):
    ev_scores_all[i].append(sl_results[k])
    posteriors[i] = self.dst.fuse(priors[i], ev_scores_all[i])
Stop condition
POMDPModule.should_stop()：达到最大步数、任务耗尽、或 country/city 已至少验证一次且 max_prob >= TRANSITION_THR 时停止。street 层不会因 transition threshold 提前停，只看任务耗尽/步数。
Action selection
如果只剩一个 task，直接选它；多个 task 时调用 policy prompt。解析失败就默认 task 0，保证流程不断。
Evidence memory
每条 evidence 的 w_scores 都加入 evidence_scores_all，posterior 每次从初始 prior 和所有累计证据重新融合，而不是只基于上一步增量。
Posterior
对候选地点的当前概率分布，例如 united states:0.51, canada:0.49。
SL weight
W(e|l)=exp(alpha * beta * (c-3) * uncertainty_factor)。大于 1 支持，小于 1 反驳，接近 1 中性。
DST fallback
当不同证据强烈支持不同候选时，把冲突质量部分转成 ignorance，避免乘法更新过度自信。
Key evidence
如果某条证据的最大权重 max_w > 1.5，会截取前 120 字放入下层 context。

4. 当前实际给 LLM 的 Prompt 模板

下面是代码里的核心 prompt 结构，省略了真实图片对象，只保留发给模型的文字。它们分别对应：生成候选、验证证据、选择下一步、给证据打分。

Hypothesize Prompt

用途：让模型先提出候选地点和检查计划。

定义：_hypothesize_prompt(image, level, context)；调用：_hypothesize() 和 predict_batch() 的 hyp_messages。 
You are a geolocation expert. {level_hint}
Return 3-5 plausible hypotheses when the image is ambiguous.
For country-level reasoning, return country names only, not continents or regions.
Do not default to United States or Canada from English text, generic roads,
vegetation, architecture, online media, or product branding alone; assign high
confidence only when there are explicit local clues.
Prior context: {context}

Analyze this image and respond with valid JSON only, no markdown fences:
{
  "hypotheses": [{"location": "<name>", "confidence": <0-1>}, ...],
  "verification_plan": [{"desc": "<what to check>", "bbox": [x,y,w,h] or null}, ...]
}

City Context

用途：country 之后进入 city 时，避免只传一个错误父节点。

定义：_context_for_level("city", result, key_evidence)；输入：country_posterior top-3。 
Country candidates: {top countries}.
Hypothesize cities within these candidate countries.
Do not introduce a city from a different country unless visible text
or a landmark directly names it.
If the country remains ambiguous, keep alternatives within the listed candidates.
Key clues: {recent high-weight evidence}

Street Context

用途：street 层在 top country/city 内细化，修复 v7 child-country conflict。

定义：_context_for_level("street", result, key_evidence)；输入：country_posterior 和 city_posterior top-3。 
Country candidates: {top countries}. City candidates: {top cities}.
Refine within these city/country candidates.
Return a street, district, or landmark;
do not append a different country unless directly visible in the image.
Key clues: {recent high-weight evidence}

Verify Prompt

用途：针对一个检查任务重新看图，产出 observation。

定义：_verify_prompt(image, task, hypotheses, level)；调用：POMDP loop 选出 task 后。 
Task: {task desc}. Focus on region [x,y,w,h]={bbox}.
Current hypotheses: {top hypotheses}
Reasoning level: {country/city/street}

Describe what you observe and how it relates to the hypotheses.
Respond with: <observation text>

POMDP Policy Prompt

用途：如果还有多个 pending tasks，让模型选最有信息量的一项。

定义：POMDPModule._make_policy_prompt(...)；返回 JSON 里的 task_index。 
You are a geolocation agent at reasoning level: {level} (step {step}).

Current belief over locations:
  {location: probability, ...}

Pending verification tasks:
  [0] desc: ..., region: ...
  [1] desc: ..., region: ...

Select the single task that would most reduce uncertainty -- i.e.,
the task most likely to produce evidence that strongly supports or
contradicts the current top hypotheses.

Respond with JSON only: {"task_index": <int>, "reason": "<short reason>"}

SL Scoring Prompt

用途：把一条 evidence 转成对每个 hypothesis 的支持强度。

定义：SLModule._make_prompt(evidence_desc, hypothesis, level)；每个 evidence × hypothesis 都会打一遍分。 
You are evaluating geographic evidence for the location hypothesis: '{hypothesis}'.
Current reasoning level: {level}.

Evidence: {evidence description}

Rate how strongly this evidence supports the hypothesis.
Respond with:
  Support Rating: <1-5>  (1=strong contradiction, 3=neutral, 5=strong support)
  Confidence: <0.0-1.0>  (your confidence in this rating)
  Reasoning: <one sentence>
设计重点：Hypothesize 负责“提出候选”，Verify 负责“看图取证”，SL scoring 负责“把文字证据量化”。不要让同一个 prompt 同时做完所有事，否则模型很容易确认偏误：先猜美国，然后后面的解释都围绕美国展开。
Prompt 进入模型的位置
MLLMClient.batch_generate() 负责普通生成；SLModule.score_many() 内部用 batch_sample_n() 做 5 次采样。
为什么要求 JSON
Hypothesize 和 policy 输出需要被代码解析。_try_parse_json() 支持纯 JSON、Markdown fenced JSON；_parse_hypothesis_payload() 还支持 wrapper 里的 general 和 cue_responses。
为什么 prompt 里反复限制美国/加拿大
v6b 诊断显示主要粗粒度错误是 North America false positives，尤其把欧洲图预测成美国。这个约束是为了降低英语/普通道路/品牌带来的默认美国偏置。
为什么 city/street 不让随便换国家
v7 证明 child 层过度自由会导致 Country-child conflict rate 暴涨到 51.84%。v8 让 child 层优先在 parent top candidates 内细化。

5. 概率论设计：为什么要这样更新

1. 先有候选集合

地理定位不是直接回归经纬度，而是在候选地点集合上做分类式推理：L = {l1, l2, ...}。

2. Prior 表示初始相信程度

MLLM 初始 confidence 不直接当真，而是经过 softmax 变成总和为 1 的概率。

3. Evidence 变成 likelihood

每条观察证据 e 对每个地点 l 都产生一个权重 W(e|l)。

4. Bayesian update

用 posterior ∝ prior × likelihood。支持某候选的证据会放大它，反驳证据会压低它。

Softmax prior
P0(l_i)=exp(score_i/T)/sum_j exp(score_j/T)。这样所有候选概率加起来等于 1。T=1.5 让分布别太尖，保留第二第三候选，减少早期过度自信。
Score cutoff
PRIOR_CUTOFF=0.6 会截断过高 confidence，避免模型一上来给 0.9 就锁死。这个是防 confirmation bias 的一部分。
SL likelihood
W(e|l)=exp(alpha * beta * (c-3) * uncertainty_factor)。c=3 中性，c>3 支持，c<3 反驳。
为什么用 exp
因为多个独立证据在 Bayes 里是 likelihood 相乘。把评分映射成指数权重后，支持/反驳可以平滑累积，而且始终为正。
Confidence alpha
alpha 是模型对自己评分的置信度。低 confidence 的证据影响小，高 confidence 的证据影响大。
Uncertainty shrink
同一 evidence 会采样 5 次。如果评分波动大，说明模型不稳定，uncertainty_factor 会把权重拉回 1，减少噪声证据影响。
Bayes update
P_t(l) ∝ P_0(l) * Π_t W(e_t|l)。代码用 log 累加再 softmax，避免很多权重相乘造成数值下溢。
DST fallback
如果多条证据互相强烈冲突，普通乘法可能错误地变得很自信。Yager DST fallback 把一部分冲突质量放到 ignorance，相当于“我不知道”，降低过度自信。
直观例子：如果候选是 France 和 United States，证据是“蓝白红旗和法语路牌”。SL 可能给 France c=5，United States c=2。Bayes 更新后 France posterior 上升，United States 下降。反过来，如果证据只是“英文广告牌”，prompt 会要求不要直接默认美国，因为英语是弱证据，不该给过强 likelihood。

6. 层级传递：v8 怎么避免 city/street 带偏

Country Posterior

保留完整国家概率分布，不只看 argmax。

City Context

传入 top-3 国家：Country candidates: A, B, C。

City Constraint

要求城市在候选国家内生成；除非图中直接出现其他国家的文字/地标。

Street Context

传入 top 国家 + top 城市，要求在这些候选内细化。

Conflict Reduced

目标是把 v7 的 51.84% country-child conflict 拉回 v6b 附近。

关键教训：v7 允许 city/street 用 embedded country 覆盖 parent country，虽然北美假阳性降了，但大量 child 输出和 parent 冲突，整体精度下降。v8 改成“冲突 child 不覆盖、不混查，直接跳过该 child geocode”。

7. 评估端 Geocode 决策树

Salvage Country

如果 pred_country 不是有效国家，尝试从 city/street 字符串里提取国家。

Try Street

如果 street 存在，用 parent country 限定查询；若 child 自带冲突国家，跳过 street。

Try City

street 失败后尝试 city，同样先检查 child-country conflict。

Try Country

城市/街道失败后，至少用国家中心附近的 geocode 结果算粗粒度距离。

Continent Fallback

全部 geocode 失败时，用 country posterior top-3 投票出 continent centroid。

street_country_qualified
街道名没有国家，评估端拼成 street, country 后查到坐标。
street_bare
带国家查询失败，裸 street 查到了。这个来源有歧义风险。
child_country_conflict
v8 新逻辑：child 自带国家和 parent country 不一致，跳过该 child，不让它覆盖 parent。
continent_fallback
最后兜底，只希望救 <2500 km 大洲级指标。

9. 关键术语备注

Hypothesize

“提出假设”。MLLM 先看整张图，列出可能地点和置信度，并给出接下来要检查的线索计划。

代码：_hypothesize_prompt / _hypothesize

Verify

“验证线索”。按 verification task 再看图，例如检查文字、路牌、建筑、植被等，并输出 observation。

代码：_verify_prompt

Update

“更新概率”。把 Verify 得到的证据转成每个候选的支持权重，然后更新 posterior。

代码：DSTModule.fuse

Hypothesis

候选地点。country 层是国家名；city/street 层是城市、街区、地标或街道。

例：Canada、Toronto、Dundas West

Prior

还没有验证证据前的初始概率。当前由 MLLM confidence 经过 temperature softmax 得到。

代码：_softmax_prior

Posterior

融合证据后的概率分布。最终输出取 posterior 最大的 MAP 候选。

例：France:0.62, Spain:0.24

Softmax

把一组分数转成总和为 1 的概率分布。temperature 越高，分布越平；越低，top 候选越尖锐。

当前：PRIOR_TEMP=1.5

POMDP

这里不是训练 RL，而是让 MLLM 根据当前 belief 选择最值得检查的下一条 verification task。

代码：POMDPModule.select_action

SL

Single-source likelihood。对一条证据分别评价它支持每个 hypothesis 的程度和置信度。

代码：SLModule.score_many

DST

Dempster-Shafer Theory。当前只作为高冲突 fallback；主路径仍是 GeoBayes 的乘法 Bayesian update。

代码：DSTModule.fuse

MAP

Maximum a posteriori。选择 posterior 最大的候选作为该层输出。

代码：best = max(posterior, key=posterior.get)

Transition Threshold

粗层到细层的停止/转移阈值。当前 TRANSITION_THR=0.55，但 country/city 至少验证一次。

代码：POMDPModule.should_stop

Geocode

把地名文本转成经纬度坐标。例如把 Paris, France 查成 (48.8566, 2.3522)。

代码：evaluate.geocode

Haversine

计算地球球面上两组经纬度之间的大圆距离。评估里的 1/25/200/750/2500 km 都基于它。

代码：evaluate.haversine

Canonicalization

地名归一化。比如 USA、California, USA、United States 都统一成 united states。

代码：canonicalize_country

Country-child Conflict

预测国家和 city/street 字符串里自带国家不一致。v7 的主要回归就是这个指标暴涨。

例：parent=united states，child=London, United Kingdom

Geocode Source

评估端记录坐标来自 street、city、country、continent fallback，便于判断问题是预测还是地名解析。

例：street_country_qualified

North America FP

预测北美但 GT 不在北美。v6b 最大错误来源，尤其是把欧洲图预测成 United States。

FP = false positive，假阳性

Continent Fallback

当所有 geocode 都失败时，用 country posterior 聚合到大洲质心，尽量保住 2500km 指标。

代码：_continent_fallback_coords

Entropy

概率分布的不确定性。候选越平均，entropy 越高；top 候选越明显，entropy 越低。后续 v9 可用它控制是否继续验证。

当前代码还没显式使用

Margin

top1 posterior 减 top2 posterior。margin 小表示模型还在犹豫，适合继续验证或 replace 候选。

例：0.52 - 0.48 = 0.04

Replace / Backtrack

GeoBayes 论文里的兜底机制。Replace 是候选不够好时重生成候选；Backtrack 是细层冲突时回到粗层重评。
